Table 5:
MPA-sim: Table of calculation cards.
CALC:AC [START=start] [STOP=stop] [STEP=step]
Calculate the AC response of the circuit from frequency start to frequency stop in steps of step.
CALC:LAYOUT
Calculate the physical layout of the circuit.
CALC:FIELD [FREQ=freq] [PHI=phi] [N=n] [SRC=src] [NORM=norm]
Calculate the radiation pattern at frequency freq by sweeping the source src at an angle phi using n steps. Results are normalised with norm.
OPTIM:alg [DIR=d] [START=start] [STOP=stop] [STEP=step] [MAXSTEP=maxstep] [SAVEVAR=savefile] [LOADVAR=loadfile] [TOLL=toll] [F j =resp j ] [AVG j =avg j ] [WGH j =wgh j ] [START j =start j ] [STOP j =stop j ] (N j =n j ) comp i :param i (min i ,max i ,step i )
Maximise (dir=1) or minimise (dir=-1) an optimisation function using the algorithm alg (Table 6). The optimisation function consists of the sum of n terms with each term given by the response resp j (Table 9) average from frequency start j to frequency stop j using average method avg j (Table 7), normalised with optimisation function term N j and weight wgh j . The response of the circuit is calculated from frequency start to frequency stop in steps of step. Optimisation is stopped when the improvement is less than toll or when maxstep steps are reached.
The parameter param i of component comp i is a`n optimisation variable that can take values between min i and max i in discrete steps of size step i . The optimised values are loaded from loadfile before optimisation and saved to savefile after each step.
Table 6:
MPAsim: Table of optimisation algorithms.
Name
Description
Step1
A gradient method where each variable is increased and decreased by a small amount, called the step length. If this results in no improvement, the step length is decreased. If the step results in an improvement, another step is taken in that direction.
Step2
Similar to the method in Step1, but after each variable is varied on its own, extrapolation steps are tried in the direction of overall improvement.
Rand
A random method, where a population (P 2 ) of circuits is randomly selected around the current solution. For each iteration, the best P 3 circuits are kept, and the other (P 2 -P 3 ) circuits are formed by randomly combining the best P 3 circuits.
Rands
A random step method, similar to the method in Step1, but where step directions are randomly chosen and the step length gradually decreases if the steps are unsuccessful and increased if the steps are successful.
External
An external program is used for optimisation, with the variables and cost function communicated using UNIX FIFO pipes.
Comp
The complex method.
ASCO
Similar to the method in External, but configured to use the optimisation program ASCO (A SPICE Circuit Optimiser).
